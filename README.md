# 68000 Starfighters

A Galaga clone set in space, for the Sega Genesis/Mega Drive, built with
[SGDK](https://github.com/Stephane-D/SGDK). Vertical-scrolling terrain and
starfield combined with classic fixed-formation enemy waves, modern
shoot'em-up player movement, and powerups dropped by special enemies.

## Prerequisites

1. **Install SGDK** (Windows release build):
   - Download the latest release from https://github.com/Stephane-D/SGDK/releases
   - Unzip it somewhere stable, e.g. `C:\SGDK`
   - Tell this project where you put it by copying
     `CMakeUserPresets.json.example` to `CMakeUserPresets.json` (gitignored,
     per-machine) and setting `SGDK_DIR` to your install path.
   - No OS-level environment variable is required — `CMakeLists.txt` forwards
     `SGDK_DIR` from the CMake cache as the `SGDK`/`GDK` environment
     variables when it invokes SGDK's own `make.exe -f makefile.gen`, so
     `CMakeUserPresets.json` is the single source of truth for the SGDK
     location on your machine.
2. **CLion** with the CMake project support (built in).
3. **A Genesis emulator** to run the built ROM — [BlastEm](https://www.retrodev.com/blastem/)
   is recommended (accurate, fast, good input latency).

## Building

This project's `CMakeLists.txt` does **not** compile the ROM itself — SGDK
uses its own GNU Makefile pipeline (`makefile.gen` + `rescomp`). CMake exists
only so CLion can index the source correctly (headers, autocomplete,
navigation). The actual build is exposed as CMake custom targets that shell
out to SGDK's `make.exe`:

- **`rom`** — release build, produces `out/rom.bin`
- **`rom_debug`** — debug build (adds `out/symbol.txt` for troubleshooting)
- **`rom_clean`** — cleans SGDK's build output

In CLion: open the CMake tool window, find these targets under the project,
and build `rom`. From the command line:

```
"%SGDK%\bin\make.exe" -f "%SGDK%\makefile.gen"
```

## Running

Point BlastEm (or another emulator) at `out\rom.bin`:

```
blastem.exe out\rom.bin
```

In CLion, this is wired up as an External Tool + Compound Run Configuration
so a single Run action builds the ROM and launches the emulator. See
`docs/` (or the project plan) for exact CLion setup steps if you're
recreating the run configuration.

Note: the `indexing_only` CMake target exists solely so CLion's code model
resolves SGDK headers and project files — it is `EXCLUDE_FROM_ALL` and never
linked. CLion may still show occasional false-positive diagnostics on
m68k-specific SGDK code (e.g. certain attributes/inline asm) since its
bundled host toolchain doesn't fully understand the m68k target; this is
expected and doesn't affect the real build, which only ever runs through
SGDK's own `make.exe`.

There is no native m68k debugger integration in CLion — use the `rom_debug`
target's `symbol.txt` with BlastEm's or Gens KMod's own debugger for
troubleshooting.

## Project Layout

- `src/` — game source (`main.c` + one module per system: player, bullet,
  enemy, formation, terrain, powerup, collision, score, sfx)
- `res/resources.res` — rescomp resource declarations
- `res/gfx/` — placeholder source PNGs referenced by `resources.res`
- `res/resources.h` — generated by rescomp on every build (not checked in)
- `out/` — build output (not checked in)
